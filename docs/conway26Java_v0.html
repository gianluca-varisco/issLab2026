ConwayLife Sprint1

Introduction

Goal: Realizzazione in Java del GAME OF LIFE DI CONWAY limitatamente al requisito R1.

Requirements

R1  Realizzare una versione in Java del gioco Life di Conway, come gioco zero-player. 
    Il gioco consiste nell’introdurre una Griglia di Celle il cui stato (cella ‘viva’ o cella ‘morta’) 
    evolve come stabilito dallle regole di ConwayLife

R2  L’utente umano deve poter:
     - specificare la configurazione iniziale della griglia del gioco	
     - vedere l’evoluzione del gioco in forma opportuna 
        (si veda Problema della vista del gioco )
     - fermare e far ripartire l’evoluzione del gioco
     - pulire (a gioco fermo) la configurazione della griglia del gioco

Che cosa intende il committente per Cell?
Per Cell, dopo aver parlato con il committente, formalizzo il seguente modello:
    
    package main.java.conway.domain;

		public interface ICell {
            /* l'entità ha la capacità di rispondere ad una sorta di query con il proprio stato (viva o morta) */
			public boolean isAlive();

            /* l'entità dovrà avere la capacità di modificare il proprio stato interno secondo un valore booleano */
			public void setStatus(boolean status);

            public void switchStatus();
		}

Che cosa intende il committente per Grid?
Il committente ha un'idea di Grid composta da Cell.
E' un'entità a due dimensioni (una volta fissate non variabili): colonne e righe.


	package main.java.conway.domain;

	public interface IGrid {
		/* l'entità deve essere in grado di restituire le proprie dimensioni */
		public int getRows();	// primitiva, non possono essere negativo -> test
    	public int getCols();	// primitiva, non possono essere negativo -> test

		/* entità deve essere in grado di restituire una sua componente */
    	public ICell getCell(int row, int col);	//primitiva


    	public int countAliveNeighbors(int row, int col);	// non primitiva, utility
    	
		/* l'entità fornisce un metodo per resettare lo stato di tutti i componenti (celle) */
		public void clear();	// non primitiva
	}

Che cosa intende il committente per Life?
Il committente non sa cos'è Life, ma mi dice che è la logica del gioco, contiene in sè le regole del gioco
Strutturalmente, Life è composta da una Griglia


	package main.java.conway.domain;

public interface LifeInterface {
	/** Calcola l'evoluzione dello stato alla generazione successiva */
    void nextGeneration();

    /** Restituisce lo stato di una cella specifica */
    boolean isAlive(int row, int col);	//non primitiva

    /** Imposta lo stato di una cella */
    void setCell(int row, int col, boolean alive);	//non primitiva

    /** Restituisce il numero di righe e colonne */
//    int getRows();
//    int getCols();
    
    /** Restituisce la Cella */
    ICell getCell(int x, int y);
    
    /** Restituisce la grid */
    Grid getGrid();
    
    /** pulisce */
    void resetGrids();
    
    /** Restituisce una rappresentazione grafica testuale della grglia*/
    public String gridRep( );
}
